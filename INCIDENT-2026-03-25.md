# Incident Report: Foreman Outage — 2026-03-25

## Summary

Foreman went down during a conversation about integrating Temporal.io and Docker. Two separate issues caused the outage, both stemming from the new `src/temporal/` scaffold that was added mid-conversation.

## Timeline

1. **Temporal scaffold added** — New files created in `src/temporal/` (worker.ts, activities.ts, client.ts, workflows.ts) and `src/index.ts` was updated to call `startTemporalWorker()` on boot.
2. **Crash loop begins** — Foreman's Node.js process crashed on startup and launchd restarted it **436 times** before the issue was diagnosed.
3. **Issue 1 fixed** — ESM import extensions and dynamic imports (see below).
4. **Issue 2 discovered** — After Foreman came back up, Claude Code sessions refused to start with: `Error: Claude Code cannot be launched inside another Claude Code session.`
5. **Issue 2 fixed** — Added `delete process.env.CLAUDECODE` at startup, rebuilt, clean restarted launchd.
6. **Service restored** — All Foreman bots responsive in Slack.

## Issue 1: ESM Import Crash Loop

### Root Cause

`src/temporal/worker.ts` used static imports without `.js` extensions:

```typescript
// BROKEN — missing .js extension, static import crashes at module resolution
import { Worker } from '@temporalio/worker';
import * as activities from './activities';
```

This project uses ESM modules, which require `.js` extensions on relative imports. Node.js failed at **module resolution time** — before any runtime code executed — so the `.catch()` in `index.ts` never caught the error.

### Fix

Changed to dynamic imports with `.js` extensions:

```typescript
// FIXED — dynamic imports are catchable, .js extensions satisfy ESM
export async function startTemporalWorker(): Promise<void> {
  const { Worker } = await import('@temporalio/worker');
  const activities = await import('./activities.js');
  // ...
}
```

Also replaced `require.resolve('./workflows')` with `import.meta.url` (ESM-compatible).

### Lesson

All imports in this codebase must use `.js` extensions. Optional subsystems should use dynamic `await import()` so the main process survives if they fail to load.

## Issue 2: CLAUDECODE Nested Session Detection

### Root Cause

A **version mismatch between the Claude Code CLI and the Agent SDK**:

- **Claude Code CLI v2.1.83** added a new anti-nesting guard: if `CLAUDECODE=1` is in the environment, it refuses to start and exits with code 1.
- **Agent SDK v0.2.70** sets `CLAUDECODE=1` on every child process it spawns — this has always been the case, and previously Claude Code accepted it without issue.
- The CLI's guard includes a bypass: if `--team-name` is passed, nested sessions are allowed. But the SDK doesn't pass `--team-name` unless the caller explicitly provides a `teamName` option, which Foreman doesn't use.

The result: the SDK spawns Claude Code with `CLAUDECODE=1` → the new CLI guard sees it → rejects it as a nested session → exits with code 1. Every `query()` call fails.

The error message was:
```
Error: Claude Code cannot be launched inside another Claude Code session.
Nested sessions share runtime resources and will crash all active sessions.
To bypass this check, unset the CLAUDECODE environment variable.
```

### Fix

Added `delete process.env.CLAUDECODE` at the top of `src/index.ts`, before any config loading or session spawning:

```typescript
// Prevent "nested session" detection when the Agent SDK spawns Claude Code.
// CLI v2.1.83 added an anti-nesting guard that checks for CLAUDECODE=1,
// but SDK v0.2.70 sets this on child processes without the --team-name bypass.
delete process.env.CLAUDECODE;
```

This prevents the env var from being inherited by Claude Code subprocesses. It's a workaround for the CLI/SDK version mismatch — a future SDK update will likely handle this properly.

### Resolution

After both fixes, a full rebuild (`npx tsc`) and clean launchd restart (`launchctl bootout` + `launchctl bootstrap`) restored all services.

## Files Changed

- `src/index.ts` — Added `delete process.env.CLAUDECODE` at startup; added `startTemporalWorker()` import and call
- `src/temporal/worker.ts` — Changed static imports to dynamic imports with `.js` extensions
